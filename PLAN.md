## Deferred / open questions

- **Multi-catalog deploy automation** ‚Äî catalog name is a session var (`01_config.sql`) and a CLI env var (`DATABRICKS_CATALOG` in `.env`); 0.1.1 lays bash + bundle foundation but isolated per-branch/PR catalogs still require manual setup. A Terraform module or `databricks bundle` target-per-PR pattern would automate this.
- **Refresh proc vs MV split** ‚Äî DECISIONS.md #5 documents the operational distinction; 0.1.1's `06_demos/02_what_is_an_mv.sql` and `03_what_is_a_table.sql` make the distinction explicit pedagogically. The S0‚ÄìS4 placement scenarios still group them as "materialized"; a 0.1.4 follow-on could add a dedicated "table-refresh" scenario.
- **Free Edition compute budget** ‚Äî bronze MV initial materialization is the heavy phase (~5‚Äì10 min). Smaller `seed_n_*` defaults already chosen but worth revisiting if Free Edition users hit quota.
- **Silver vtransaction_fact downstream consumers** ‚Äî 0.1.1 added `investments.vtransaction_fact` to back `gold_pd_consolidated.vpd_transaction_book` (DECISIONS.md #15). It's currently consumed only at the consolidated layer. Per-team transaction views (`team_pd_*.vtransaction_fact`) would let team gold also consume transactions; deferred until a use case appears.

- PowerBI semantic model consumption ‚Äî that's `pbi-metadata-capture` and PBIP fixtures.
- Cost-modelling automation ‚Äî out of scope.
